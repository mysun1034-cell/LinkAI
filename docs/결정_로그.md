### ADR-03 · 착수 전 결정 5건 심층분석 (Chris "분석만 더 깊게" 선택)
- **사건**: 첫 슬라이스 착수 vs 분석 심화 선택지에서 Chris가 **"분석만 더 깊게"** 선택.
- **결과**: README §11.7 열린 질문 5개를 결정 가능한 수준으로 심화(`docs/착수전_결정_분석.md`). 5개 전용 자문 렌즈 병렬 딥다이브 + 웹 검증. 제안(모두 Chris 최종 결정 대기):
  1. **배포 형태 = 로컬퍼스트(재정의)** — raw_content 온디바이스. B(SaaS)/C(하이브리드)는 유일 차별점(프라이버시)을 버리며 PIPA·COGS까지 인수. **기각**.
  2. **타깃 = Git 솔로 개발자** + `SUPERSEDED 감사궤적`을 aha 웨지로(대체재는 '현재 스냅샷'뿐).
  3. **진척도 = 체크리스트 우선 혼합** — "완료조건 X/Y 검증", 단일 % 강등(단일실패점). 교정 로그 = 해자 훅.
  4. **로컬 LLM = HyperCLOVA X SEED + Mistral Small 3.2 조합**, bake-off로 확정. **EXAONE 3.5/4.0 둘 다 비상업(NC) 독립 확인 → 상용 후보 탈락**. SEED 상업 무료(월 1000만 MAU 이하) 독립 확인.
  5. **PIPA = crypto-shredding + Local 기본 + opt-in 별도동의** + **변호사 게이트**(§7.2 무삭제 vs 파기·삭제권 충돌). crypto-shredding의 PIPA 파기 인정 여부는 미확인 → 변호사 확인 필요.
- **왜**: 배포 형태가 스택·보안·가격을 다 가르는 상위 결정. 코드 착수 전 근거 확보.
- **근거**: `docs/착수전_결정_분석.md`. 커밋 `043c8bf`. (주: 딥다이브 워크플로의 adversarial verify 단계는 스크립트 버그로 미실행 → load-bearing 라이선스 사실은 본 세션이 독립 재확인.)

### ADR-04 · [중대 방향 제안] 진짜 의도 = 영어학습 웹 (LINK AI English)
- **사건**: Chris가 속내를 밝힘 — *"이걸 만드는 이유는 팀프로젝트를 해야 해서인데, 진심은 영어 학습 웹에 적용하는 것."* 별도 기획서 **`LINK_AI_English_웹기반_영어학습_개발기획서_v1.0.docx`** 제출.
- **LINK AI English 요지**: 범용 LINK AI를 **영어학습으로 1:1 이식**한 버전. Project Twin→**Learner Twin**, Current State→현재 실력, Evidence-based Progress→**Evidence-based Mastery**, Change Timeline→Growth Timeline, Attention Engine→오답 기반 학습 추천. 타깃=한국 초3~중3(MVP=중1 문법·독해, 2022 개정 교육과정). Skill Graph(PREREQUISITE/PART_OF/RELATED/CONFUSABLE_WITH/TRANSFER_TO/CURRICULUM_MAP), Learner Twin 지표(Mastery/Evidence Confidence/Stability/Freshness/Misconception/Transfer/Trend), 증거 이벤트(FIRST_TRY_CORRECT·CORRECT_AFTER_HINT·WRONG·DELAYED_REVIEW_CORRECT + 가중치), 화면(TODAY/Skill Map/Growth Timeline/Weakness Map + 교사·학부모·관리자·검수자), AI Tutor(규칙·캐시·DB 우선, LLM은 힌트·피드백·튜터링).
- **분석 판정(권고)**: **영어학습판을 팀프로젝트로 통합하는 것이 타협이 아니라 업그레이드.** 이유:
  - 범용판 **최약점(증거→완료조건 의미 판정)이 "문항 정답/오답 = 객관 신호"로 해소** → 제품 심장이 최약점에서 최강점으로.
  - 이미 좁은 버티컬(중1 영어) → '범용이 wedge 죽인다' 문제 소멸.
  - **로컬 에이전트·File System Access·raw_content·남의 코드 PIPA 지옥이 전부 사라짐**(평범한 웹앱) → 팀프로젝트 스코프에 적합.
  - 콜드스타트 없음(진단평가로 시작). 대체재(Claude/Cursor)가 안 하는 교육 도메인.
  - **엔진(Twin/Current State/Evidence/Freshness) 1:1 전이** → LINK AI 아키텍처 고민이 안 버려짐(기획서 §2.1이 이미 매핑).
- **새 리스크(범용판의 '추측금지'에 대응)**: 콘텐츠·교육학 정확성 — Skill Graph 선후관계·2022 개정 교육과정 매핑·문항 품질·Mastery 가중치/복습 주기(매직넘버)는 근거(교육과정·간격반복 SM-2/Leitner 연구) 확보 후 결정. CareLink의 추측금지·근거주석 규율을 그대로 적용.
- **상태**: **방향 미확정 — Chris 최종 결정 대기.** (범용 유지 vs 영어학습 통합 vs 비교 후 결정)
